EigenDA Orbit SDK Fork

Overview

This is a fork of the Arbitrum Orbit SDK that has been customized to work with EigenDA as the data availability layer for Arbitrum Orbit chains.

The EigenDA Orbit SDK allows developers to deploy, configure, and manage Arbitrum Orbit chains with EigenDA integration for secure, decentralized data availability.

Key Modifications

1. EigenDA Default Configuration

The SDK has been updated with default values optimized for EigenDA integration:

  • Updated WASM module root to point to EigenDA-compatible Nitro build
  • Modified NodeConfig to include EigenDA-specific parameters
  • Added EigenDARollupManager field to deployment configuration
  • Enhanced SequencerInbox contract to support EigenDA blob verification

2. Custom Rollup Creators

Implemented custom rollup creator contracts specifically designed for EigenDA integration:

  • Added EigenDA-specific rollup creator contracts for testnet and production networks
  • Updated token bridge creator contracts to work with EigenDA rollups
  • Maintained compatibility with the Arbitrum Orbit interface while adding EigenDA capabilities

3. Package Namespace Changes

Updated package namespace and related metadata:

  • Changed package name from @arbitrum/orbit-sdk to @eigenda/orbit-sdk
  • Updated documentation references to reflect EigenDA integration
  • Modified examples to use the new package namespace

4. Multiple Batch Posters Support

Enhanced batch poster management for improved reliability:

  • Modified batch poster representation from single address to array of addresses
  • Updated transaction request handling to support multiple batch posters
  • Enhanced integration tests to validate multiple batch poster functionality

Using the EigenDA Orbit SDK

Install the SDK via npm: 

npm install @eigenda/orbit-sdk

For detailed documentation and examples, refer to the GitHub repository.
